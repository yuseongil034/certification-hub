# 컴활 2급 실기 핵심 요약

---

## 1. 시나리오

변경셀 · 결과셀 찾기 → **이름정의 먼저** → 시나리오 추가

- 이름정의 안 하면 요약 보고서에 셀 주소로 표시됨 (이름으로 안 나옴)
- 요약 보고서 생성 시 결과셀 범위 정확히 지정할 것
- 변경셀은 여러 개 지정 가능 (Ctrl 클릭)

---

## 2. 데이터 표

결과값 셀에 **=참조 입력 먼저** → 영역 잡기 → 데이터 표 실행

- 영역은 =참조 셀 포함해서 잡을 것
- 행 입력셀 / 열 입력셀 구분
  - 표 상단(가로)에 값 → 행 입력셀
  - 표 왼쪽(세로)에 값 → 열 입력셀

---

## 3. 정렬

조건 **순서대로** 추가 (위쪽이 우선순위 높음)

- ⚠️ 부분합 사용할 거면 **반드시 정렬 먼저**
- 사용자 지정 정렬 시 기준 열 · 정렬 기준 · 순서 세 가지 모두 확인

---

## 4. 부분합

**기준 열 정렬 먼저** → 전체 영역 잘 잡기 → 부분합 실행

- 중첩 시 두 번째부터 **"현재 부분합 대치" 체크 해제**
- 윤곽 기호(1·2·3)로 요약 레벨 조절 가능
- 제거할 땐 **"모두 제거"** 클릭

---

## 5. 통합

영역 잡기 → 입력할 표 & 참조 표 **형태 동일**해야 함

- 형태 다를 경우 → **레이블(행/열 제목) 기준으로 통합 옵션 체크**
- 그래도 다르면 영역을 맞게 조정할 것
- 원본 데이터 연결 체크 시 → 원본 변경되면 결과도 자동 반영
- ⚠️ 참조 범위 추가할 때 **"추가" 버튼** 눌러야 목록에 등록됨

---

## 6. 매크로

- 기록 시작 후 **다른 셀 먼저 클릭**하고 행동할 것
  → 셀 선택 위치까지 매크로에 기록되기 때문
- 데이터는 **미리 입력 후** 매크로 적용할 것
  → 원하는 서식으로 적용하기 쉬움
- 저장 위치는 **현재 통합 문서**로 설정
  → 다른 곳에 저장하면 해당 파일에서 불러와지지 않음
- 단추(버튼)에 연결 시 **양식 컨트롤** 사용 (ActiveX 컨트롤과 혼동 주의)
- 실행 후 결과 확인 필수 → 잘못 기록됐으면 삭제 후 다시 기록

---

## 7. 차트

- 데이터 범위 수정 · 추가 시 범위 잘못 잡지 않도록 조심
- 특정 요소 수정 → **차트 더블클릭(편집모드 진입)** 후 요소 클릭
  → 차트 전체 클릭(1번) vs 편집모드 진입(2번) 구분할 것
- 축 제목, 범례, 데이터 레이블 등 각 요소는 **개별 클릭 후 서식** 적용
- 차트 이동 시 **개체로 이동 vs 새 시트로 이동** 구분
  → 문제에서 시트 지정하면 반드시 확인
- 보조 축 필요 시 해당 계열 클릭 → **데이터 계열 서식 → 보조 축** 선택

## 8. 함수

### 📌 기본 규칙

- 수식 다음에 오는 문자는 **무조건 `""` (쌍따옴표)** 로 묶기
- 문자와 문자 사이는 **무조건 `&` (앰퍼샌드)** 로 연결
- 공란(빈 값) 표현은 `""` (띄어쓰기 없는 쌍따옴표)
- ⚠️ 범위가 고정되어야 하는 경우 **반드시 절대참조 (`$`)** 사용
  → 자동채우기(드래그) 시 범위 밀리면 결과 전부 틀림 (F4로 빠르게 고정)

---

### D 함수 (조건 만족 함수)

> DSUM, DAVERAGE, DCOUNT 등

```
D함수(전체범위, 필드, 조건범위)
```

- ⚠️ **유일하게 필드(열 제목)까지 범위에 포함**해서 사용
- 조건범위 구성 방식은 **고급 필터와 동일**

---

### 수학 함수

- **ROUND / ROUNDUP / ROUNDDOWN**: 반올림 · 올림 · 내림
  - 자릿수 양수 → 소수점 아래 / 음수 → 소수점 위
  - ⚠️ 자릿수 잘 보고 판단할 것
- **MOD(수, 나누는수)**: 나머지 반환 → 결과값은 **숫자**

---

### 통계 함수

**RANK.EQ**
```
RANK.EQ(자기자신, 범위, 순서)
```
- 범위는 **반드시 절대참조**
- 순서: 내림차순 → `0` 또는 생략 / 오름차순 → `1`

**LARGE / SMALL**
```
LARGE(범위, k)  /  SMALL(범위, k)
```
- k번째는 숫자 직접 입력 또는 **셀 참조** 모두 가능

---

### 텍스트 함수 (LEFT / MID / RIGHT)

- 결과값은 숫자처럼 보여도 **무조건 문자 취급**
- IF문 등에서 비교할 때 **반드시 `""` 붙이기**
  - 예: `IF(LEFT(A1,1)="3", ...)` ✅
- 몇 개를 제외하고 가져오고 싶을 때 → **LEN에서 빼기(-) 활용**
  - 예: `LEFT(A1, LEN(A1)-2)` → 뒤 2글자 제외하고 가져오기

> ⚠️ MOD 결과는 숫자 / LEFT·MID·RIGHT 결과는 문자 — 혼동 주의!

---

### COUNTIF / SUMIF

**COUNTIF** — 조건을 만족하는 셀 **개수** 세기
```
COUNTIF(범위, 조건)
```

**SUMIF** — 조건을 만족하는 셀의 값 **합계** 구하기
```
SUMIF(조건범위, 조건, 합계범위)
```

> ⚠️ SUMIF는 인수 3개! 조건범위와 합계범위가 다를 수 있음

**예제 데이터**

| 지역 (A열) | 매출 (B열) |
|---|---|
| 서울 | 100 |
| 부산 | 200 |
| 서울 | 150 |

- `COUNTIF(A1:A3, "서울")` → `2` (서울 개수)
- `SUMIF(A1:A3, "서울", B1:B3)` → `250` (서울 매출 합계)
- `SUMIF(A1:A3,"서울",B1:B3) / COUNTIF(A1:A3,"서울")` → `125` (서울 평균)

**조건 작성 규칙**

| 조건 형태 | 작성 방법 | 예시 |
|---|---|---|
| 셀 참조만 단독 | `""` 없이 그냥 참조 | `COUNTIF(A:A, B1)` |
| 문자 직접 입력 | `""` 로 묶기 | `COUNTIF(A:A, "서울")` |
| 비교연산자 + 숫자 | `""` 로 전체 묶기 | `COUNTIF(A:A, ">=10")` |
| 비교연산자 + 셀 참조 | `"연산자"&셀` 로 연결 | `COUNTIF(A:A, ">="&B1)` |

- SUMIF ÷ COUNTIF → **조건부 평균** 계산 가능

---

### 조건 함수

**IF**
```
IF(조건, 참일때 값, 거짓일때 값)
```
- 중첩 IF → 거짓 자리에 계속 IF 삽입

**IFS**
```
IFS(조건1, 값1, 조건2, 값2, ..., TRUE, 기본값)
```
- 마지막에 해당하는 조건 없을 때 → `TRUE` 입력 후 기본값 작성

**IFERROR**
```
IFERROR(수식, 에러일때 표시할 값)
```

---

### CHOOSE / SWITCH

**CHOOSE**
```
CHOOSE(인덱스번호, 값1, 값2, ...)
```
- **숫자 인덱스일 때만 사용 가능**
- 문자 값은 `""` 로 묶기
- ⭐ 함수 마법사(fx) 활용하면 값 채우기 편함

**SWITCH**
```
SWITCH(식, 값1, 결과1, 값2, 결과2, ..., 기본값)
```
- **숫자 · 문자 모두 사용 가능**
- 식과 값이 일치하면 해당 결과 반환

---

### WEEKDAY

```
WEEKDAY(날짜, 형식)
```
- ⚠️ 기본형(형식 생략 또는 1) → **1 = 일요일**, 7 = 토요일

### VLOOKUP / HLOOKUP

**VLOOKUP** — 표에서 값을 **세로(열)** 방향으로 찾기
```
VLOOKUP(찾을값, 범위, 열번호, 0)
```

**HLOOKUP** — 표에서 값을 **가로(행)** 방향으로 찾기
```
HLOOKUP(찾을값, 범위, 행번호, 0)
```

**사용 조건**
- ⚠️ 찾을값은 반드시 범위의 **첫 번째 열(VLOOKUP) / 첫 번째 행(HLOOKUP)** 에 있어야 함
- 구하려는 값은 찾을값 기준 **오른쪽(VLOOKUP) / 아래쪽(HLOOKUP)** 에만 가능
- 범위 잡을 때 찾을 열/행이 **맨 왼쪽/맨 위** 에 오도록 잡기

**열번호 / 행번호**
- 범위의 첫 번째 열(행)이 1번
- 원하는 값이 몇 번째 열(행)인지 세서 입력

**마지막 인수**
- `0` → 정확히 일치 (시험에서 거의 항상 0 사용)
- `1` → 근사값 일치 (거의 안 나옴)

**VLOOKUP 예제** — 데이터가 세로로 나열된 경우

| 이름 (A열) | 점수 (B열) |
|---|---|
| 홍길동 | 90 |
| 김철수 | 85 |

→ `VLOOKUP("홍길동", A1:B3, 2, 0)` = `90`  
→ "홍길동"이 첫 번째 열, 점수는 두 번째 열 → 열번호 = **2**

**HLOOKUP 예제** — 데이터가 가로로 나열된 경우

| | A열 | B열 |
|---|---|---|
| 1행 | 홍길동 | 김철수 |
| 2행 | 90 | 85 |

→ `HLOOKUP("홍길동", A1:B2, 2, 0)` = `90`  
→ "홍길동"이 첫 번째 행, 점수는 두 번째 행 → 행번호 = **2**

---

### ✅ 핵심 비교 정리

| 함수 | 결과값 | 비고 |
|---|---|---|
| LEFT / MID / RIGHT | **문자** | 숫자처럼 보여도 문자, `""` 붙이기 |
| MOD | **숫자** | 혼동 주의 |
| CHOOSE | 숫자 인덱스만 | SWITCH는 문자도 가능 |
| VLOOKUP / HLOOKUP | 찾을값 기준 오른쪽(아래) | 첫 열(행)에 찾을값 있어야 함 |
| RANK.EQ 범위 | 절대참조 필수 | 드래그 시 밀림 방지 |
